HTML

  • The head element contains information about the webpage.
  • The head element contains the title, metadata, character settings of the webpage.
  • The body element represents the visible content shown to the user.
  • The body element contains links, lists, buttons, images and more for the user to interact with
  • The head contains less imformation than the body in most cases, but both elements are equally important.
  • HTML can be considered a mark-up language.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS can either be written in HTML document, or created in a speparate document.
  • Class attributes share a CSS rule to any element by assigning the rule to a class attribute with a class selector.
  • If CSS document is in a separate document, it needs to be linked via the head section in the HTML document.

Git

  • git status: Checks what branch we are currently on
  • git checkout -b branch-name: Creates a new branch and switches to it
  • git checkout branch-name: Switches to branch
  • git branch -d branch-name: Delete branch
  • git add -A: Commit all files on working branch
  • git pull origin main: Pulls main branch into project
  • git push origin branch-name: Pushes branch to main project
  • Always update the new branch with the main branch before any push requests.
  • Handle pull requests in GitHub

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Chrome DevTools is a great tool for debugging.
  • In JavaScript, one can declare functions, make if statements and for loops, call functions and debug.
  • JavaScript uses OOP principles.
  • The inspect tool in Chrome allows one to view the different code making up a webpage.
